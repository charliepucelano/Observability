# Recent Corrective Fixes (Do Not Re-diagnose)

This is a running changelog of fixes and mitigations applied to the homelab. When analyzing logs and metrics, please assume these issues are already resolved and do not propose them as new solutions unless the metrics show the mitigation has explicitly failed.

- **[2026-07-12] Price Crawler - `firebase_auth.sh` DB Variable Bug**: Moved `export DB=...` above the token cache check block. Previously, if the token was loaded from cache, the script would return early without exporting `DB`. This caused downstream scripts like `recheck_alerts.sh` to fail with `ERR_INVALID_URL` because `fetch` calls to `${DB}` evaluated to a relative path (`/priceAlerts/...`). Fixed in `agent-skills` and requires syncing.
- **[2026-07-10] Loki Dashboard Log Level Regex Fix**: Fixed the Loki dashboard query `loki-dashboard-improved.json` that was broadly filtering `|~ "(?i)${level:regex}"`. This was causing Grafana to wrongly highlight `info` logs as `error` if they happened to contain the word "error" in the message body. Updated the query regex to strictly match log level structures like `level=error` or `[error]`.
- **[2026-07-10] Blackbox Paperless Django DisallowedHost Fix**: Configured the Prometheus `blackbox` exporter to use a dedicated `http_paperless` module with the HTTP `Host: paperless.leyforge.com` header when probing the Paperless target. This resolves `[django.security.DisallowedHost]` warnings generated in the Django logs.
- **[2026-07-10] Grafana Dashboard BOM Encoding Fix**: Removed a UTF-8 BOM from `loki-dashboard-improved.json` which was causing dashboard provisioning to fail with `invalid character 'ï' looking for beginning of value`.
- **[2026-07-05] Weekly Analysis Parse Error**: Fixed a Unicode em-dash character (`U+2014`) on line 166 of `weekly-analysis.ps1` that caused PowerShell to throw a `Missing closing '}'` parse error, preventing the entire script from loading. This broke both the Sunday 09:00 auto-trigger and the `/analyze` manual command since Jul 4.
- **[2026-07-05] /dump Clipboard Fix**: Rewrote the `/dump` handler in `telegram-bot-listener.ps1` to run the dump generation inline instead of via `Start-Process`. The NSSM service runs in Session 0, so `Set-Clipboard` in a child process was setting the clipboard for Session 0 (invisible to the user's interactive desktop).
- **[2026-07-05] Perplexity Watchdog Selector Fix**: Replaced the single `.prose` CSS selector in `analyze.sh` watchdog with a multi-strategy approach: tries `.prose`, `[class*="prose"]`, `[class*="markdown"]`, `[class*="answer"]`, `[class*="response"]`, `[class*="rich-text"]`, then falls back to finding the largest text block containing typical AI answer patterns (markdown headers, bold text, "Action Items", etc.). Perplexity frequently changes their DOM class names.
- **[2026-07-05] NSSM ContainerWatchdog/RAGWatcher AppDirectory**: Both services were crashing on startup (<1500ms) because `AppDirectory` registry key was unset, causing NSSM to default to `C:\windows`. Set `AppDirectory` to `D:\scripts` for both services. This was generating 15,339 warning/error events per week in Windows Event Log.
